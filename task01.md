**角色**：资深前端开发工程师 (Next.js & Canvas 图形处理专家)
**任务**：在现有“老约翰工作台”基础上，架构“实用小工具”板块，并完成第一个工具“图片二维码合成”的开发。

**环境确认**：
1. 项目基于 Next.js App Router + Tailwind CSS。
2. **关键资源**：项目根目录的 `public` 文件夹下已存在一个名为 `qrcode.png` 的文件。

**具体需求**：

### 1. 首页架构升级 (`app/page.tsx`)
* **分区设计**：在现有的6大功能模块下方，添加一个明显的分割线或标题（如 "🛠️ 运营工具箱"）。
* **小工具列表**：
    * 在下方创建一个新的 Grid 区域。
    * **视觉降权**：相比上方的主业务卡片，小工具卡片要设计得更紧凑（Compact Card）。
    * **样式建议**：左侧是小图标（24px），右侧是工具名称（加粗）和极简描述（单行灰色小字）。高度控制在 80px 左右。
    * **交互**：点击卡片通过 `<Link>` 跳转到 `/tools/[toolId]`（内部路由）。

### 2. 工具箱路由架构
* 创建 `app/tools/config.ts`：
    * 定义工具配置数组。目前添加第一个工具：
        * `id`: 'qr-composer'
        * `name`: '二维码合成器'
        * `description`: '自动在图片右下角添加官方二维码'
        * `icon`: 使用 Lucide React 的 'QrCode' 图标。
* 创建动态路由页面 `app/tools/[toolId]/page.tsx`：
    * 读取 `params.toolId`。
    * 根据 ID 渲染对应的组件（目前仅需处理 'qr-composer'，其他显示“开发中”）。
    * **布局要求**：页面左上角必须有一个“< 返回工作台”的按钮。

### 3. 核心开发：二维码合成组件 (`QrComposer`)
* **组件位置**：建议在 `app/tools/_components/QrComposer.tsx` 或类似位置。
* **功能逻辑**：
    * **输入**：提供一个简洁的拖拽/点击上传区域，接受用户上传底图。
    * **处理 (Canvas)**：
        1.  加载用户上传的图片。
        2.  加载项目中的静态资源 `/qrcode.png`（注意：这是关键路径）。
        3.  使用 HTML5 Canvas 将底图绘制在画布上。
        4.  **合成算法**：将二维码绘制在画布的**右下角**。
            * *大小控制*：二维码宽度设为底图宽度的 15%（或至少 150px，避免过小）。
            * *边距控制*：距离右边缘和下边缘各保留底图宽度的 5% 作为 Padding。
    * **输出**：
        * 左侧/上方显示操作区。
        * 右侧/下方实时显示合成后的 Canvas 预览图。
        * 提供一个醒目的“下载合成图片”按钮。

**交付物**：
请提供更新后的 `app/page.tsx`，新建的 `app/tools/config.ts`，`app/tools/[toolId]/page.tsx` 以及核心组件 `QrComposer.tsx` 的完整代码。
